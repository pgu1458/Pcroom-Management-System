# 관리자 PC (Admin Server)


피씨방 통합 관리 시스템의 **서버 겸 관리자 클라이언트**입니다.  
프로그램을 실행하면 TCP 서버(9000 / 9001 / 9002 포트)가 자동으로 시작되며, 모든 클라이언트의 요청을 처리하는 중심 역할을 합니다.


---


## 주요 기능


### 로그인 (CounterLogin)


아이디와 비밀번호로 로그인하면 Member.db의 `role` 컬럼 값에 따라 화면이 분기됩니다.


- **관리자** → 매출 차트 및 메뉴 관리 화면(Admin)으로 이동
- **알바** → 좌석·주문·채팅·회원 관리 화면(CounterAl)으로 이동
- **손님** → "키오스크를 이용해주세요" 안내 후 로그인 거부


로그인 처리는 서버를 거치지 않고 로컬 DB를 직접 조회하는 방식이라 서버 부하 없이 즉시 응답합니다.


---


### 좌석 관리


![좌석 관리](screenshots/admin_seats.png)


50개 좌석의 이름 / 이용시간 / 남은시간을 카드 형태로 실시간 표시합니다. 사용 중인 좌석은 파란색, 비어 있는 좌석은 검은색으로 구분되어 한눈에 현황을 파악할 수 있습니다.


- 1초 타이머(Timer)가 이용시간을 증가시키고 남은시간을 차감하며 UI를 갱신합니다
- 사용자 PC에서 로그인이 발생하면 서버 이벤트를 통해 해당 좌석이 즉시 활성화됩니다
- 키오스크 충전 완료 시 해당 회원의 남은시간 라벨이 서버 이벤트로 즉시 갱신됩니다
- 좌석 카드를 클릭하면 해당 회원의 상세 정보가 ListView에 표시됩니다
- 화면 상단에 현재 이용 중인 좌석 수가 실시간으로 표시됩니다 (예: 12/50석)


---


### 채팅 관리


![채팅 관리](screenshots/admin_chat.png)


사용자 PC에서 문의 메시지가 수신되면 해당 좌석 카드가 노란색으로 강조 표시되어 관리자가 즉시 인지할 수 있습니다. 좌석을 클릭해 이용자를 선택하면 해당 이용자와의 채팅 이력이 표시되며, 하단 입력창에서 답장을 전송합니다.


- 채팅 이력은 `[HH:mm] 이용자: ...` / `[HH:mm] 관리자: ...` 형식으로 시간과 함께 출력됩니다
- 사용자 PC는 1.5초 폴링으로 답장을 수신하므로, 전송 후 최대 1.5초 내에 상대방 화면에 답장이 표시됩니다
- 여러 좌석에서 동시에 문의가 들어와도 각 좌석별로 이력이 독립적으로 관리됩니다


---


### 회원 관리


![회원 관리](screenshots/admin_members.png)


Member.db의 전체 회원 목록을 DataGridView로 조회합니다. 전화번호 뒷자리 입력 후 검색 버튼을 누르면 해당 번호가 포함된 회원만 필터링됩니다.


- 그리드 셀을 직접 수정하고 저장 버튼을 누르면 DB에 즉시 반영됩니다
- 회원 삭제 시 seat_number를 확인해 현재 이용 중인 회원은 삭제가 제한됩니다
- 회원가입 요청은 9000 포트로 수신되며, 서버에서 아이디 중복 확인 후 DB에 저장됩니다
- 로그아웃 수신 시 해당 회원의 `time` 컬럼에 남은시간을 저장하고 `seat_number`를 0으로 초기화합니다


---


### 상품 목록 및 주문 관리


![상품 및 주문 관리](screenshots/admin_food_orders.png)


Food.db의 전체 메뉴를 상단 DataGridView에서 조회합니다. 셀을 더블 클릭하면 직접 수정할 수 있고, 저장 버튼으로 DB에 반영합니다.


사용자 PC에서 주문이 수신되면 하단 주문 목록 그리드에 자동으로 추가됩니다. 이 때 서버에서 Food.db의 해당 메뉴 `inventory`(재고량)를 차감하고 `sale`(판매량)을 증가시킵니다. 매출 데이터는 sales.db에도 함께 기록됩니다. 주문 목록에서 불필요한 내역을 선택해 삭제할 수 있습니다.


---


### 음식 재고 관리


![재고 관리](screenshots/admin_inventory.png)


메뉴별 입고량 / 재고량 / 판매량을 조회하고 직접 수정할 수 있습니다. 각 속성에 따라 데이터가 정렬되며, 셀을 직접 클릭해 수정한 뒤 저장 버튼을 누르면 반영됩니다. 재고 부족 메뉴를 빠르게 확인하고 입고량을 조정하는 용도로 활용됩니다.


---


### 매출 차트 (Admin)


![매출 차트](screenshots/admin_sales.png)


월과 일을 선택하면 해당 기간의 매출을 막대 차트로 확인할 수 있습니다. 일을 선택하지 않으면 해당 월 전체의 일별 매출을 보여주고, 날짜까지 선택하면 해당 날의 시간대별 매출로 드릴다운됩니다. 차트 하단에는 조회 기간의 총 매출 금액이 함께 표시됩니다.


- 차트는 `System.Windows.Forms.DataVisualization`의 `Chart` 컨트롤을 사용해 구현했습니다
- sales.db를 집계 쿼리로 조회해 데이터를 가져오며, 월 변경 시 차트가 즉시 갱신됩니다

## 서버 구조 (TcpServer)


싱글톤 패턴으로 구현된 TCP 서버입니다. `Form1` 생성자에서 `TcpServer.Instance.Start()`가 호출되며, 프로그램 종료 시 `Stop()`으로 모든 리스너와 스레드를 정리합니다.


```csharp
TcpServer.Instance.Start();   // Form1 생성자에서 자동 호출
TcpServer.Instance.Stop();    // 프로그램 종료 시 자동 호출
```


각 포트는 `TcpListener`로 독립적으로 리슨하며, 클라이언트 연결 수락 후 `Thread`를 생성해 비동기적으로 처리합니다. UI 갱신이 필요한 경우 `Invoke`를 통해 메인 스레드에서 안전하게 처리합니다.


**9000 포트 처리 내용**

- 로그인 요청 수신 시 DB에서 아이디/비밀번호 검증
- 검증 성공 시 비어 있는 좌석을 자동 탐색해 배정, seat_number 업데이트
- 회원가입 요청 수신 시 아이디 중복 확인 후 DB INSERT
- 로그아웃 수신 시 남은시간 DB 저장 + seat_number 0으로 초기화
- TIME_REQ 수신 시 DB에서 현재 time 조회 후 TIME_RES 응답
- CHAT 수신 시 메시지 저장, 관리자 UI에 알림 (좌석 노란색 표시)
- CHAT_POLL 수신 시 해당 사용자의 대기 중인 답장 여부 확인 후 응답


**9001 포트 처리 내용**

- ORDER 수신 시 주문 내역 파싱
- Food.db에서 각 메뉴의 inventory 차감, sale 증가
- sales.db에 날짜·시간대·금액 INSERT
- 관리자 UI 주문 목록 그리드에 즉시 추가
- ACK 응답 전송


**9002 포트 처리 내용**

- 연결 즉시 전체 50석 현황을 `SEATS|번호:상태:잔여초,...` 형식으로 푸시
- LOGIN 수신 시 DB 검증 후 로그인 성공/실패 응답
- CHARGE 수신 시 DB의 time 컬럼 업데이트, 이용 중인 경우 관리자 UI 남은시간 즉시 갱신
- 연결 끊김 감지 시 해당 키오스크 클라이언트 정리

## 프로젝트 구조


```
Admin/
├── Program.cs          # 진입점 (Form1 실행)
├── Form1.cs            # 메인 폼 (TcpServer 시작/종료, 화면 전환)
├── CounterLogin.cs     # 로그인 화면 (role 기반 권한 분기)
├── CounterAl.cs        # 알바 화면 (좌석/주문/채팅/회원/재고 관리)
├── Admin.cs            # 관리자 화면 (매출 차트, Food.db 직접 수정)
├── TcpServer.cs        # TCP 서버 싱글톤 (9000/9001/9002 멀티포트)
├── Checksum.cs         # 체크섬 생성 및 검증 유틸
├── Member.db           # 회원 DB (SQLite)
├── Food.db             # 음식 메뉴/재고 DB (SQLite)
└── sales.db            # 매출 DB (SQLite)
```

## 실행 방법


1. Visual Studio 2022에서 프로젝트 열기
2. NuGet 패키지 설치
   ```
   System.Data.SQLite
   System.Windows.Forms.DataVisualization
   ```
3. 빌드 후 실행 (관리자 PC를 가장 먼저 실행해야 합니다)
4. Member.db에서 `role = '관리자'` 또는 `role = '알바'` 계정으로 로그인
