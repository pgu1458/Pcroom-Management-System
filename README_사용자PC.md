# 사용자 PC (User Client)


피씨방 통합 관리 시스템의 **사용자 PC 클라이언트**입니다.  
각 자리에 설치되어 로그인부터 이용시간 카운트다운, 게임 런처, 음식 주문, 관리자 채팅까지 피씨방 이용에 필요한 모든 기능을 제공합니다.


---


## 주요 기능


### 로그인 



<img width="800" height="450" alt="로그인" src="https://github.com/user-attachments/assets/57794853-736b-4ba1-8b81-81b49bb4591c" />




**로그인 흐름**

1. 서버에 `LOGIN|id|pw` 전송
2. 서버에서 DB 검증 후 빈 좌석 자동 배정
3. `LOGIN_OK|name|remainSec|seatNum` 수신 시 이름 / 남은시간 / 좌석번호를 팝업으로 표시
4. 확인 버튼 클릭 후 메인 화면(UserPcMain)으로 전환


로그인 실패 시 서버에서 전달된 사유(아이디 없음, 비밀번호 불일치, 이미 로그인 중 등)를 팝업으로 표시합니다. 엔터 키로도 로그인이 가능하도록 구현했습니다.




### 회원가입





<img width="800" height="500" alt="회원가입" src="https://github.com/user-attachments/assets/2e144563-315a-4a5a-a5f7-98c59ff8e8be" />





**회원가입**은 별도 팝업에서 진행됩니다. 이름 / 생년월일 / 아이디 / 비밀번호 / 전화번호를 입력하고 가입완료 버튼을 누르면 `REGISTER|name|id|pw|birth|phone|손님` 메시지가 서버로 전송됩니다. 서버에서 아이디 중복 여부를 확인하고 DB에 저장한 뒤 `REGISTER_OK` 또는 `REGISTER_FAIL|reason`으로 응답합니다.



프로그램 실행 시 전체 화면으로 고정된 로그인 화면이 표시됩니다. 아이디와 비밀번호를 입력하고 로그인 버튼을 누르면 9000 포트로 `LOGIN|id|pw` 메시지가 전송됩니다.

### 메인 화면 (UserPcMain)


<img width="800" height="500" alt="메인창" src="https://github.com/user-attachments/assets/b2fb36de-75e4-460e-b416-b9b392af6a1c" />




로그인 성공 후 표시되는 메인 화면으로, 이용 정보 / 게임 런처 / PC 설정 / 음식 주문 / 채팅 / 로그아웃 기능이 하나의 화면에 통합되어 있습니다.


**이용 정보 표시**

- 우측 상단에 이름 / 좌석 번호 / 남은 이용 시간(HH:MM:SS) 표시
- 1초 Timer로 남은시간을 로컬에서 카운트다운
- 30초마다 서버에 `TIME_REQ|id`를 전송해 서버 기준 시간과 동기화 (키오스크 충전 즉시 반영)
- 시간이 만료되면 알림 팝업을 표시하고 로그인 화면으로 복귀


**로그아웃**

- 로그아웃 버튼 클릭 시 현재 남은시간을 포함한 `LOGOUT|id|remainSec` 메시지를 서버로 전송
- 서버에서 DB에 남은시간을 저장하고 좌석을 반환
- 로그인 화면으로 복귀

### 음식 주문 (Menu)


<img width="800" height="500" alt="image" src="https://github.com/user-attachments/assets/c33f284f-977b-4dce-b62b-47c08328cbc4" />





주문 버튼을 누르면 Menu 폼이 팝업으로 실행됩니다. 상단에 카테고리별 탭이 있고, 각 탭에서 메뉴 버튼을 클릭하면 하단 장바구니(ListView)에 추가됩니다.


| 카테고리 | 메뉴 수 |
|----------|---------|
| 면류 | 17종 |
| 밥류 | 8종 |
| 분식 | 8종 |
| 스낵 | 10종 |
| 음료 | 16종 |


- 같은 메뉴를 다시 클릭하면 수량이 자동으로 1 증가합니다
- 장바구니 항목을 더블 클릭하면 해당 메뉴가 삭제됩니다
- 장바구니에 변동이 생길 때마다 총액이 실시간으로 계산되어 표시됩니다
- 주문 전송 버튼 클릭 시 `ORDER|orderId|items|totalPrice` 메시지에 체크섬을 추가해 9001 포트로 전송합니다
- 서버로부터 `ACK|orderId|OK`를 수신하면 주문번호와 총액을 팝업으로 안내하고 장바구니를 초기화합니다


주문 전송에 9001 포트를 별도로 사용하는 이유는 9000 포트에서 처리 중인 채팅·시간 동기화 등의 트래픽과 분리해 처리 지연을 방지하기 위함입니다.

### 문의 채팅 (ChatForm)


<img width="800" height="500" alt="메신저" src="https://github.com/user-attachments/assets/62a1d291-f210-4869-b797-94f225572b67" />




문의 버튼을 누르면 ChatForm 팝업이 실행됩니다. 하단 입력창에 메시지를 입력하고 전송 버튼(또는 엔터 키)을 누르면 `CHAT|userId|message` 메시지가 9000 포트로 전송됩니다.


- 전송 후 `CHAT_OK`를 수신하면 채팅 이력에 내 메시지가 시간과 함께 표시됩니다
- 1.5초 주기로 `CHAT_POLL|userId`를 서버에 전송해 관리자 답장을 수신합니다 (`Timer` 활용)
- `CHAT_REPLY|msg` 수신 시 채팅 이력에 관리자 메시지가 추가됩니다
- `CHAT_EMPTY` 수신 시에는 별도 동작 없이 다음 폴링을 대기합니다
- ChatForm을 닫으면 폴링 타이머가 정지됩니다

## TCP 클라이언트 구조 (TcpClientService)


싱글톤 패턴으로 구현되어 있으며, 9000 포트와 9001 포트 두 개의 연결을 관리합니다.


- **9000 포트 연결** : 로그인 시 생성, 로그아웃 시 종료. 로그인·로그아웃·시간 동기화·채팅 등 모든 일반 통신에 사용됩니다
- **9001 포트 연결** : 음식 주문 전송 시 연결, 응답 수신 후 종료. 주문 처리를 9000 포트와 분리해 독립적으로 처리합니다


```csharp
public string ServerIp { get; set; } = "192.168.0.6";
```

## 체크섬 (Checksum.cs)


모든 메시지는 전송 전에 4자리 16진수 체크섬을 계산해 메시지 앞뒤에 추가합니다. 수신 측에서는 체크섬 검증 후 통과한 메시지만 처리합니다. 이를 통해 네트워크 전송 중 발생할 수 있는 데이터 손상을 감지합니다.

## 프로젝트 구조


```
UserPC/
├── Program.cs              # 진입점
├── Form2.cs                # 메인 폼 (전체화면 고정, ESC 종료)
├── UserPcLogin.cs          # 로그인 / 회원가입 화면
├── UserPcMain.cs           # 메인 화면 (시간 카운트다운, 게임, 설정, 주문, 채팅, 로그아웃)
├── Menu.cs                 # 음식 주문 폼 (카테고리 탭, 장바구니)
├── ChatForm.cs             # 관리자 채팅 폼 (폴링 기반 수신)
├── TcpClientService.cs     # TCP 클라이언트 싱글톤 (9000/9001 포트)
├── Checksum.cs             # 체크섬 생성 및 검증 유틸
└── Images/                 # 메뉴 이미지 리소스 폴더
```
