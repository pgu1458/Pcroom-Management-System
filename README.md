#  PC방 키오스크 — 좌석 실시간 연동 + 사용자 UI

> 5인 팀 프로젝트 **PC방 통합 관리 시스템** 중 **키오스크 클라이언트 전체**를 담당
> 50석 실시간 좌석 동기화 + 회원 로그인 + 시간 충전 + 3종 결제 모달

시연 영상 : https://youtu.be/eAQMOYcSDC8

## 프로젝트 소개

5인 팀이 함께 만든 PC방 통합 관리 시스템(`Pcroom-Management-System`)에서, 저는 **고객이 직접 사용하는 키오스크 클라이언트와 그 통신 프로토콜**을 처음부터 끝까지 담당했습니다.

키오스크는 매장 입구에 놓이는 단독 PC로, 손님이 자기 자리를 보고 시간을 충전하는 화면입니다. 단순한 UI가 아니라 — 매장 내 50대의 사용자 PC 상태가 **실시간으로** 좌석 배치도에 반영되어야 하고, 회원 인증·충전·결제가 카운터 서버와 즉시 동기화되어야 합니다. 이걸 위해 TCP 포트 하나(9002)를 따로 두고 키오스크 전용 통신 프로토콜을 직접 설계했습니다.

## 주요 기능

###  50석 좌석 실시간 동기화 (핵심)

매장 PC 50대의 사용 상태와 **남은시간(초 단위)**이 키오스크 좌석 배치도에 실시간으로 반영됩니다. 손님이 키오스크 앞에서 빈자리를 즉시 확인할 수 있도록, 다음 3중 메커니즘을 두었습니다.

**1) 서버 푸시 — 사용자 PC 온/오프 즉시 반영**
사용자 PC가 켜지거나 꺼지면, 카운터 서버가 **요청 없이도** 키오스크로 `SEATS` 패킷을 보내옵니다. 키오스크는 받은 즉시 좌석 배치도 색상과 사용중 표시를 갱신합니다.

```csharp
// 백그라운드 수신 루프에서 SEATS 패킷 자동 처리
if (line.StartsWith("SEATS|"))
{
    var seats = ParseSeats(line);
    OnSeatsUpdated?.Invoke(seats);   // UI 갱신 이벤트 발행
}
```

**2) 1초 카운트다운 타이머 — 남은시간 자동 감소**
서버가 매초 보내는 것이 아니라, 키오스크가 **자체적으로 매초 남은시간을 1씩 감소**시켜 라벨에 표시합니다. 네트워크 트래픽을 줄이면서도 화면은 실시간으로 흐릅니다. 서버 푸시가 오면 그때 정확한 값으로 동기화됩니다.

**3) 5초 재연결 타이머 — 끊김 자동 복구**
연결이 끊겨도 5초마다 자동으로 재접속을 시도하고, 재접속 즉시 서버가 보내주는 최신 `SEATS`로 화면이 복구됩니다.

### 🔐 회원 로그인 (9002 프로토콜)

키오스크 ID/PW 입력 → 카운터 서버 9002 포트로 `LOGIN|id|pw` 전송 → 응답 파싱.

| 단계 | 메시지 |
|------|--------|
| 키오스크 → 서버 | `LOGIN|user1|pass1234` |
| 서버 → 키오스크 (성공) | `LOGIN_OK|홍길동|3600` |
| 서버 → 키오스크 (실패) | `LOGIN_FAIL` |

로그인 성공 시 `KioskTcpClient.Instance`의 세션(`CurrentUserId`, `CurrentUserName`, `CurrentRemainTime`)에 자동 저장되어 이후 화면에서 그대로 사용합니다. 5초 응답 타임아웃, 동시 송신 방지를 위한 SemaphoreSlim 락, `TaskCompletionSource` 기반 비동기 응답 매칭으로 안정성을 확보했습니다.

###  시간 충전 + 3종 결제 모달

로그인하면 5가지 충전 옵션이 나오고, 결제 수단을 골라 모달에서 결제를 진행합니다.

**충전 옵션**

| 금액 | 시간 |
|------|------|
| 50,000원 | 60시간 |
| 30,000원 | 35시간 |
| 10,000원 | 10시간 |
| 3,000원 | 3시간 30분 |
| 1,000원 | 1시간 |

선택된 시간은 정규식 파싱(`(\d+)\s*시간`, `(\d+)\s*분`)으로 초 단위로 변환되어 서버에 전송됩니다.

**3종 결제 모달**

| 수단 | UserControl | 동작 |
|------|-------------|------|
| 신용카드 | `Credit` | 30초 타이머 + 결제 완료 버튼 |
| 카카오페이 | `KaKao` | 동일 |
| 네이버페이 | `Npay` | 동일 |

세 결제 화면 모두 `Pay` 모달 Form 안에서 분기되어 표시됩니다.

```csharp
Pay payForm = new Pay(num);   // num: 1=신용카드 2=카카오 3=네이버
DialogResult payResult = payForm.ShowDialog();
if (payResult != DialogResult.OK) return;  // 취소·타임아웃이면 충전 안 함
```

**30초 타임아웃 + 시각적 경고**
결제 모달은 30초 카운트다운을 띄우고, 남은시간에 따라 라벨 색이 바뀝니다.

| 남은시간 | 색상 |
|---------|------|
| 15초 이상 | 기본 |
| 5~15초 | 🟡 노랑 |
| 5초 미만 | 🔴 빨강 |
| 0초 | 자동 취소 |

결제 완료 시 서버에 `CHARGE|id|addSeconds` 전송, 응답으로 새 남은시간을 받아 표시한 뒤 **TCP 연결 해제 → 로컬 세션 초기화 → 프로그램 자동 재시작**으로 다음 손님을 위해 첫화면으로 돌립니다.

### 🔌 9002 통신 프로토콜 (직접 설계)

키오스크는 카운터 서버 9002 포트 한 채널만 사용하며, 모든 메시지는 **UTF-8 텍스트 + `\n` 종단**의 단순한 파이프(`|`) 구분 포맷입니다.

| 방향 | 메시지 | 의미 |
|------|--------|------|
| 키오스크 → 서버 | `LOGIN|id|pw` | 로그인 요청 |
| 키오스크 → 서버 | `CHARGE|id|addSeconds` | 시간 충전 요청 |
| 서버 → 키오스크 | `LOGIN_OK|name|remainSec` | 로그인 성공 + 세션 데이터 |
| 서버 → 키오스크 | `LOGIN_FAIL` | 로그인 실패 |
| 서버 → 키오스크 | `CHARGE_OK|newRemainSec` | 충전 성공 + 새 남은시간 |
| 서버 → 키오스크 | `SEATS|1:1:3600,2:0:0,...,50:0:0` | 좌석 50개 상태 일괄 푸시 (좌석번호:사용중(1/0):남은시간초) |

**SEATS 포맷 하위호환**
초기 버전은 `SEATS|1:1,2:0,...`처럼 남은시간 없이 보냈었는데, 파싱 코드에 3번째 필드가 없어도 0으로 처리하는 fallback을 두어 서버·키오스크를 동시 업데이트하지 않아도 깨지지 않게 했습니다.

**비동기 요청-응답 매칭**
백그라운드 수신 루프 하나가 모든 메시지를 받기 때문에, 송신측은 `TaskCompletionSource`를 미리 만들어두고 수신 루프가 응답 메시지 prefix(`LOGIN_OK|`, `LOGIN_FAIL`, `CHARGE_OK|`)로 매칭해 `TrySetResult`로 깨워주는 방식을 썼습니다. 5초 안에 응답이 안 오면 타임아웃 처리.

## ▶ 빌드 / 실행

### 요구사항
- Windows 10 이상
- .NET 6 SDK
- Visual Studio 2022 또는 `dotnet` CLI

### 실행 순서

1. **카운터 PC** 먼저 실행 (팀 저장소의 `test` 프로젝트) — TCP 9000/9001/9002 리스닝
2. **키오스크 PC**에서 `Kiosk_Sum.csproj` 빌드 후 실행
   - `KioskTcpClient.cs` 의 `ServerIp` 를 카운터 PC의 IPv4 주소로 수정
   - 기본 포트: 9002
3. 키오스크 화면이 뜨면 첫화면에서 "좌석조회" 누르면 실시간 50석 배치도, "충전" 누르면 로그인 → 충전 → 결제

```csharp
// KioskTcpClient.cs
public string ServerIp { get; set; } = "192.168.0.6";  // ← 카운터 PC IP로 수정
public int ServerPort { get; set; } = 9002;
```

### 방화벽

카운터 PC에서 9002 인바운드 허용:

```bash
netsh advfirewall firewall add rule name="Pcroom-9002" dir=in action=allow protocol=TCP localport=9002
```

## ⚠ 한계 및 개선 방향

**서버 IP 하드코딩**
`KioskTcpClient.cs` 에 카운터 PC IP가 상수로 박혀 있어 환경이 바뀌면 코드를 고쳐야 합니다. 별도 설정 파일(`config.json`)이나 첫 실행 시 입력 받는 화면을 추가하면 좋습니다.

**좌석번호 라벨 이름 불규칙**
Designer에서 좌석 50개 라벨이 `label2`, `label6`, `label12` 식으로 불규칙하게 명명되어, 초기화 시 텍스트로 찾아 캐싱하는 우회를 썼습니다. 처음부터 `seatLabel_1` ~ `seatLabel_50` 같은 규칙으로 명명했어야 했습니다.

**결제 후 프로세스 재시작 방식**
충전 완료 후 첫화면으로 돌아가기 위해 `Environment.Exit(0)` + 새 프로세스 시작 방식을 사용했습니다. 빠르고 확실하지만, TCP 연결도 매번 재수립해야 하므로 비효율적입니다. 화면 컨트롤만 교체하는 식으로 리팩토링하면 더 깔끔합니다.

**시간 동기화 신뢰성**
키오스크는 1초마다 자체 카운트다운하므로, 시스템 시계가 부정확하면 표시 시간이 실제 서버 시간과 어긋날 수 있습니다. 주기적으로 서버에서 SEATS를 다시 받아 보정하지만, 명시적인 시계 동기화 메시지가 있으면 더 정확합니다.

**중복 클릭·세션 방어**
충전 중 중복 클릭은 버튼 비활성화로 막지만, 멀티 키오스크 환경에서 같은 계정이 동시 로그인하는 경우는 막지 않습니다. 서버 측에서 세션 단일화가 필요합니다.

**Esc 즉시 종료**
모든 폼에서 Esc 키로 즉시 종료되는데, 결제 중에도 Esc로 빠져나갈 수 있어 의도치 않은 종료가 가능합니다. 결제 진행 중에는 Esc를 막거나 확인 다이얼로그를 띄우는 것이 안전합니다.
